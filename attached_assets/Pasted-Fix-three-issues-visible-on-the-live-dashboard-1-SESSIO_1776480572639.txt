Fix three issues visible on the live dashboard:

1. SESSION / NAME BUG — shows "default-user" instead of real name:
The userId stored in session is "default-user" as a fallback.
When a user completes onboarding and POSTs to /auth/login with 
their name, that should become their session userId permanently.

In the login flow (wherever POST /auth/login is called after 
the name input on the login screen):
- Store both userId (slug version e.g. "amy-harris") AND 
  displayName (raw input e.g. "Amy Harris") in the session:
  req.session.userId = slugify(name)
  req.session.displayName = name

- Add displayName to GET /auth/me response:
  res.json({ userId: req.session.userId, displayName: req.session.displayName })

- Update the dashboard greeting to use displayName:
  "Good evening, Amy." not "Good evening, default-user."

2. TRAVELERS NOT SHOWING — "No travelers added yet":
The travelers added during onboarding are being saved with 
the session userId at that moment. But if the session userId 
changes between onboarding and dashboard load, the query 
finds nothing.

Fix: ensure the userId is set BEFORE any travelers are created 
during onboarding. The POST /auth/login must complete and set 
the session cookie before Step 1 of onboarding makes any 
API calls to POST /travelers.

In the onboarding component, make the flow:
1. User enters name → POST /auth/login → wait for 200 response
2. ONLY THEN start onboarding step 1 and make traveler API calls
3. All subsequent API calls will use the now-established session

Also check: does the onboarding POST /travelers include 
credentials: "include" on the fetch? If not, the session 
cookie won't be sent:
fetch("/api/travelers", {
  method: "POST",
  credentials: "include",  // ← this is critical
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify(data)
})

Apply credentials: "include" to ALL fetch calls in the 
frontend — travelers, preferences, properties, loyalty, 
summary, personality, reviews. Every single one.
